# 0003 — 配分3戦略と「コアは均質」という隠れた前提

日付: 2026-07-21

## 状況

soc-sign-epinions で block / reversed / interleave の3戦略を実測した直後、「blockとreversed, interleaveをそれぞれ教えて」とリクエスト。実測では interleave が idle を 32.6% → 0.1% に消したのに compute は 4.3% しか縮まず、その説明で「早いコアと遅いコアが存在するのね」という気づきがあった。

## 診断

- 3戦略の「配り方」自体は未整理だった（実装は済んでいたが、絵として説明できる状態ではなかった）
- idle% という診断指標が「コアは均質」を暗黙に仮定していること、M4（P4+E6）ではその前提が崩れることが新しい気づきだった

## 対応

レッスン `lessons/0003-batch-strategies.html` を作成。24頂点・重さ末尾偏重の縮図で、戦略と worker 数を切り替えてバッチ構成（帯）とガントチャートを見るシミュレータ（`assets/scheduler.js`、再利用可能）。理想化シミュと実測の食い違い（reversed はシミュでは改善、実測では悪化）を教材の中で明示。早見表 `reference/batch-strategies.html` も作成。

## 今後への示唆

- 学習済み: block の idle の発生機構 / reversed のクリティカルパス問題 / interleave のストライド構成
- 次の候補: P/E コア非対称の切り分け実験（w=4 P コア固定）/ LPT・work stealing との接続 / busy 膨張のもう1つの仮説（キャッシュ競合）の検証方法
- シミュレータは「均質コア仮定」を意図的に置いている。P/E コアを持つ拡張版が次のレッスンの素材になり得る
