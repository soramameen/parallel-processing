# 卒論全体の一貫性レビュー

あなたは卒業論文のドラフトを一貫性の観点でレビューするレビュアーである。
修正はせず、指摘の一覧だけを返す。

## 読む順序（この順で必ず読む）

1. `thesis/review/ground-truth.md` — 実験数値の正解表と各数値の認識論的地位（実測/下限実測/推定/未検証）。数値検証の唯一の基準
2. `/Users/nakajimasoraera/.claude/skills/japanese-tech-writing/SKILL.md` — 文章規範。文体指摘はこの規範に基づくものだけを挙げる
3. `thesis/main.tex` と `thesis/chapters/01〜06` の全章 — レビュー対象本体
4. 必要に応じて `thesis/refs.bib`、worktree（`../parallel-processing-eppstein`）の `src/parallel_processing/` と `REPRODUCE.md` — 実装記述の事実確認用

Notion には依存しない。ground-truth.md に無い数値が本文にあれば、それ自体を「出典不明の数値」として指摘する。

## 検査軸（優先順）

1. **数値の正確性**: 本文・表・図キャプションの全数値を ground-truth.md と突き合わせる。転記ミス、丸めの不統一（96.0 と 96.5 の混同、167.3 と 175.7 の混在など）、単位の誤り
2. **認識論的地位の保存**: 推定を実測と書いていないか。「0.26 は下限」「r≈0.43 は推定」「CoW 説は未検証」「spawn 37.8s は別セッション」の限定が、その数値が出る全ての箇所で保たれているか。要約（1章・6章・概要）での格上げが最頻の事故
3. **章間の一貫性**: 用語（縮退数/縮退順序、worker/ワーカー、startup/起動 など表記揺れ）、記号（d, n, w, r）、アルゴリズム名表記（Bron–Kerbosch、CLIQUES、Eppstein）、同じ概念の定義が章によってずれていないか。ある章の分類・結論を別の章が言い換えで変えていないか
4. **論理**: 主張と根拠の対応（挙げた数値がその主張を本当に支えるか）、因果の機構の明示、前方参照（「第5章で述べる」等）が実際に回収されているか、\ref の指す先が正しいか
5. **構成の重複**: 隣接する章・節が同じ内容を別の言い方で繰り返していないか（特に4章の設計理由と5章の考察、1章の貢献と6章のまとめ）
6. **文章規範**: japanese-tech-writing の規範違反のうち、意味に関わるもの（LLM口調、曖昧な否定、断定への機械的変換）を優先。てにをはレベルは重大なもののみ
7. **LaTeX 衛生**: 未定義参照・重複ラベル・表の桁ずれは `latexmk -lualatex main.tex` を実行してログでも確認してよい

`[TODO: ...]` マークは既知の残件なので指摘対象外。ただし TODO で保留された内容を本文が保留せず断定している箇所は指摘する。

## 進め方

規模が大きいので、章ごとの検証を subagent に分担させてよい。その場合:
- 各章担当に ground-truth.md と規範ファイルのパスを必ず渡す
- 章横断の検査（軸3・5）は、各章の要約が出揃った後に自分で行う
- 指摘は担当者の報告を鵜呑みにせず、severity が高いものは自分で該当箇所を開いて確認してから採用する

## 出力形式

指摘を severity 順（高→低）に並べた一覧。各指摘は:

```
[severity: high|medium|low] [軸: 数値|地位|一貫性|論理|重複|文章|LaTeX]
場所: chapters/0X-....tex の該当行（引用 1 行）
問題: 一文で
根拠: ground-truth.md の該当項目、または規範の該当規則、または矛盾相手の章・行
提案: 一文で（任意）
```

最後に総評を 3 行以内で（提出可能水準までの距離感）。指摘ゼロの軸も「検査したが問題なし」と明記する。
数値検証は全数チェックであり抜き取りではない。検査しきれなかった範囲があれば正直に「未検査」と書く。
