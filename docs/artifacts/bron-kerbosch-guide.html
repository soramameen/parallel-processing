Bron–Kerbosch アルゴリズム入門

グラフの「極大クリーク」を効率よく列挙する再帰的探索アルゴリズムを、図と疑似コードでやさしく解説します。

1. クリークって何？

グラフとは「頂点（点）」と「辺（点同士を結ぶ線）」からなる構造です。

  • クリーク（clique）: 互いにすべて辺で結ばれている頂点の集まり
  • 極大クリーク（maximal clique）: それ以上頂点を追加できないクリーク
  • 最大クリーク（maximum clique）: 頂点数が最も多いクリーク
例: 頂点 {1,2,3} が互いに繋がっていて、どこにも追加できないなら、それは極大クリークです。サイズが最も大きい極大クリークが最大クリークです。

2. Bron–Kerbosch で何ができる？

Bron–Kerbosch はすべての極大クリークを列挙するアルゴリズムです。1973年に Coenraad Bron と Joep Kerbosch によって考案されました。

「最大クリークを求めたい」場合は、極大クリークを列挙してその中から最大のものを選びます。ただし、大きなグラフでは列挙数が爆発するので、途中で「これ以上大きなクリークは作れない」部分木を切る「枝刈り」を入れると効率よく最大クリークが求められます。

3. 3つの箱 R, P, X

アルゴリズムは再帰的に動作し、各呼び出しで3つの頂点集合を持ちます。

  • R: 今作っているクリークに確定で入っている頂点
  • P: まだ追加候補になっている頂点
  • X: 除外済みの頂点
終了条件: P と X が両方空になったら、R は極大クリークなので出力します。

4. 基本形のアルゴリズム

algorithm BronKerbosch1(R, P, X):
    if P is empty and X is empty:
        report R as a maximal clique

    for each vertex v in P:
        BronKerbosch1(
            R ∪ {v},
            P ∩ N(v),
            X ∩ N(v)
        )
        P := P \ {v}
        X := X ∪ {v}

N(v) は頂点 v の隣接頂点の集合です。

動きのイメージ

  1. P から頂点 v を1つ選ぶ
  2. v を R に入れ、v と繋がっている頂点だけを新しい P にする
  3. 再帰的に探索
  4. 探索が終わったら v を P から外し、X に入れる

5. pivot による高速化

基本形は非極大クリークも探索してしまい、無駄が多いです。pivot を使うと分岐数を減らせます。

ポイント: pivot u とその隣接頂点は後で調べられるので、今はスキップできます。つまり、P \ N(u) だけを探索すれば十分です。 
algorithm BronKerbosch2(R, P, X):
    if P is empty and X is empty:
        report R as a maximal clique

    choose a pivot vertex u in P ∪ X

    for each vertex v in P \ N(u):
        BronKerbosch2(
            R ∪ {v},
            P ∩ N(v),
            X ∩ N(v)
        )
        P := P \ {v}
        X := X ∪ {v}

Tomita 版では、pivot として「P と重なりが最も少ない頂点」を選ぶことで、再帰呼び出し数を大幅に削減します。

6. 頂点順序による高速化

さらに、グラフの退化次数順（degeneracy ordering）で頂点を並べると、疎なグラフで非常に速くなります。SNS グラフなどの実世界グラフでは、この version がよく使われます。

algorithm BronKerbosch3(G):
    P = V(G)
    R = empty
    X = empty

    for each vertex v in a degeneracy ordering of G:
        BronKerbosch2(
            {v},
            P ∩ N(v),
            X ∩ N(v)
        )
        P := P \ {v}
        X := X ∪ {v}

7. 具体例

次のようなグラフを考えます。

1 ─ 2 ─ 3 ─ 4 ─ 5
│ │
└─────────┘

このグラフの極大クリークは {1,2,5}, {2,3}, {3,4}, {4,5} です。Bron–Kerbosch はこれらを重複なく列挙します。

8. なぜ並列化できる？

Bron–Kerbosch は再帰的に独立な部分問題を作ります。例えば、最初の分岐で「頂点1を含む極大クリーク」「頂点2を含む極大クリーク」…は互いに干渉しません。

そのため、各部分問題を別の CPU コアに割り当てて並列に探索できます。Python なら multiprocessing を使うのが一般的です。

注意: グラフが非対称だと、一部の部分問題だけ重くなり「負荷偏り」が起きます。これは並列化の課題であり、評価のネタにもなります。

9. まとめ

  • Bron–Kerbosch はすべての極大クリークを列挙する再帰アルゴリズム
  • 3つの集合 R, P, X を使って探索状態を管理する
  • pivot で無駄な分岐を減らせる
  • degeneracy ordering で疎なグラフを高速に処理できる
  • 部分問題が独立なので並列化も可能
