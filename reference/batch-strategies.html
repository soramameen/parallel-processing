早見表 — バッチ配分の3戦略

前提: 重い頂点は縮退順序の末尾に固まる（workload 実測）。n = 頂点数、K = バッチ数。

戦略バッチの作り方狙いと弱点実装
block 先頭から連続に batch_size 個ずつ 単純。重いバッチが最後に配られ、終盤に idle が出る eppstein_parallel.py
reversed 順序を逆にしてから連続に区切る（重い方を先に配る） 終盤の待ちは減る。ただし最重バッチ1個がクリティカルパスになり、それより速くなれない eppstein_parallel_reversed.py
interleave バッチ k = indices[k::K]（K 個おきのストライド） どのバッチも軽重が混ざり、重さがほぼ均一。idle はほぼ消える eppstein_parallel_interleave.py

実測の要点（soc-sign-epinions, w=8, batch=64, repeat 3 最小値, 2026-07-21）

block 25.9秒 (idle 32.6%) / reversed 28.5秒 (idle 23.5%) / interleave 24.8秒 (idle 0.1%)。

idle を消しても 4.3% しか縮まない。M4 の P/E コア非対称により「worker の稼働時間 ≠ 仕事量」で、idle% は均質コアを仮定した指標だから。busy 秒の合計は 139秒 → 198秒 に膨張。
